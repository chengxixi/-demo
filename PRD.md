# 反馈管理系统 PRD（更新版）

## 1. 背景与目标

当前系统用于统一管理多渠道用户反馈、售后退换货信息、工单处理、紧急异常闭环和竞品信息沉淀。此次更新目标是将原型从基础反馈列表扩展为可用于团队评审和流程演示的完整本地/线上预览版本。

核心目标：
- 统一反馈清单字段、模板字段和新增反馈字段口径。
- 支持反馈合并、明细查看、批量流转和人工复核。
- 建立 P0/P1 紧急异常处理看板、详情页和流程闭环。
- 建立工单池列表与工单详情页，支持继续流转或关闭。
- 完善竞品信息登记表字段和竞品新增校验。
- 将首页菜单命名为“仪表盘”，贴合业务表达。

## 2. 用户角色

- 管理员：可查看全部反馈、异常、工单和竞品信息。
- 产品经理：关注产品线反馈、异常处理、工单流转和竞品信息。
- 海外反馈人员：关注海外渠道反馈、异常、站点和竞品数据。
- 国内反馈人员：关注国内渠道反馈、售后和工单闭环。

## 3. 功能范围

### 3.1 仪表盘

左侧菜单“首页”调整为“仪表盘”。

要求：
- 菜单切换、页面刷新后仍保持“仪表盘”文案。
- 不改变原有路由结构。

### 3.2 反馈清单

反馈清单展示字段需覆盖：
- 编号
- 地区
- 数据来源
- 设备类型
- 品牌
- 内部型号/料号
- 销售型号
- 订单号
- 快递单号（退换货）
- 用户反馈（客户对话/退货反馈）
- 用户评价翻译（AI 自动翻译）
- 图片补充
- 视频补充
- 问题反馈时间
- 问题回答/处理方案
- 一级职能划分
- 二级问题场景分类
- 三级具体问题
- 异常级别
- 创建方式
- 处理去向
- 状态
- 反馈人
- 处理人
- 操作

已支持能力：
- 新增反馈。
- 批量导入。
- 我的模板设置。
- 筛选。
- 多站点选择。
- 待人工复核。
- 合并反馈展示。
- 反馈详情页。
- 批量转工单、转异常、转需求、转 Q&A。

### 3.3 反馈详情

反馈详情不使用右侧抽屉卡片，改为主区域完整详情页。

要求：
- 点击“详情”进入独立详情页样式。
- 提供“返回列表”按钮。
- 单条反馈详情展示并可编辑分类、处理去向、状态、处理人、用户反馈和 AI 翻译。
- 合并反馈详情中，分类、退换货进度应归属到每条明细中，不作为合并单公共字段。
- 合并明细可查看单条、移出合并。

### 3.4 我的模板设置

模板字段需参考反馈清单字段，保持口径一致。

字段包括：
- 地区
- 数据来源
- 设备类型
- 品牌
- 内部型号/料号
- 销售型号
- 订单号
- 快递单号（退换货）
- 问题反馈时间
- 图片补充
- 视频补充
- 创建方式
- 异常级别
- 处理去向
- 状态
- 反馈人
- 一级职能划分
- 二级问题场景分类
- 三级具体问题
- 用户反馈（客户对话/退货反馈）
- 用户评价翻译（AI 自动翻译）
- 问题回答/处理方案
- 备注

不包含：
- 默认处理人

### 3.5 工单池

工单池列表页不展示工单流程。

列表字段：
- 工单 ID
- 关联反馈
- 处理状态
- 当前处理人
- 责任部门
- SLA 剩余时间
- 处理结果
- 关闭原因
- 处理去向
- 操作

工单详情页：
- 展示工单流程。
- 展示当前处理人、责任部门、SLA、处理状态、处理去向、关闭原因、处理结果。
- 支持动作：
  - 转异常
  - 转 Q&A
  - 转需求
  - 直接关闭
- 操作后详情页和列表同步更新。

### 3.6 紧急异常处理

左侧菜单与页面标题统一为“紧急异常处理”。

列表范围：
- 默认只展示 P0/P1 紧急异常。
- P2/P3 仅作为统计观察。

异常编号规则：
- 异常等级 + 年月日 + 流水号
- 示例：P1-20260611-008

异常列表字段与表现：
- 异常编号
- 异常等级
- 状态
- 临时方案倒计时
- 触发来源
- 创建人
- 负责人
- 地区/站点
- 产品线/型号
- 影响范围
- 响应时限
- 临时处置
- 详情和添加处置操作

超时规则：
- P0/P1 从转入异常开始显示临时方案倒计时。
- 超时且未输出临时方案的记录置顶。
- 超时记录需明显标红，包括红色边框、左侧警示条、淡红背景、高对比超时标签。

筛选条件：
- 地区
- 产品线
- 站点
- 创建人
- 异常等级
- 状态
  - 默认未归档
  - 待响应
  - 处理中
  - 已归档
- 时间类型
  - 全部时间
  - 年度
  - 季度
  - 月度
- 时间范围
- 其他相关信息关键词

默认规则：
- 已归档异常默认不展示，只有筛选“已归档”时展示。

异常详情页：
- 展示流程：
  - 临时方案处理
  - 根因分析
  - 整改措施
  - 验证结果
  - 关闭归档
- 展示时限提醒卡。
- 展示影响范围。
- 展示处理团队、处理人、状态、反馈来源、触发来源、站点、产品线、内部型号等。
- 可保存流程。
- 可关闭归档。

新建异常：
- 字段包括：
  - 异常 ID
  - P 级
  - 地区
  - 处理人
  - 站点
  - 产品线
  - 内部型号
  - 触发来源
  - 影响范围
  - 异常问题
- 地区联动：
  - 国内默认处理人：刘海州，默认站点：京东
  - 海外默认处理人：陈总，默认站点：Amazon
- 保存后生成新的紧急异常卡片。

### 3.7 竞品分析

竞品信息页保留原有卡片区，不额外插入重复卡片。

新增竞品弹窗：
- 渠道类型：
  - 海外
  - 国内
- 海外渠道必须填写 ASIN。
- 国内渠道可不填写 ASIN。
- 竞品标签：
  - 品牌竞品，默认勾选
  - ODM 竞品，可同时勾选

登记字段需参考《竞品信息登记表》：
- 竞品类型
- 内部竞品
- 品牌
- 产品名称
- 产品型号
- 产品类型
- 产品定位
- 上市时间
- 销售平台
- 销售区域
- 竞品链接
- 信息来源
- 信息采集日期
- 信息采集人
- 信息状态
- 官方售价
- 到手价
- 币种
- 备注

产品类型根据产品线变化：
- 八电极秤：八电极体脂秤、四电极体脂秤、体重秤
- 体脂秤：体脂秤、体重秤、智能健康秤
- 筋膜枪：迷你筋膜枪、专业筋膜枪、热敷筋膜枪

必填字段保存时需校验。

## 4. 非功能要求

- 当前版本通过注入脚本增强打包后页面，不重做原应用路由结构。
- 本地预览地址：`http://127.0.0.1:4180/`
- GitHub Pages 部署后提供团队可访问链接。
- 页面脚本需通过语法检查。
- 页面需兼容当前桌面宽度下的主要操作流程。

## 5. 当前限制

- 当前为原型预览版本，数据主要为前端内存数据。
- 新增/修改记录刷新页面后不保证持久化。
- 竞品新增暂不写回原页面已有卡片数据源，仅保证新增弹窗字段和校验完整。
- 异常、工单、反馈流转为演示闭环，后续需接入后端接口后固化状态。

- 将合并反馈、异常归档、工单流转做成后端持久状态。
- 增加权限控制和操作日志。
- 增加导出 Excel、附件上传、图片/视频预览能力。
- 将当前注入脚本能力回迁到正式前端代码结构中。
